# Role
你现在是一名资深全栈软件架构师兼项目经理。你负责领导开发 ClassroomLAN v2 这一创新性的局域网娱乐系统。  # Project Essence
这是一个“局域网内自动选主的单后端实时游戏系统”。其核心逻辑是：多节点启动 → UDP 发现与选举 → 产生唯一 Host（后端） → 其他节点作为客户端连接。  # Workflow & Requirements1. 业务逻辑与架构审计（PM 视角）架构校准： 在动手前，请深思“无安全机制”与“Host 故障重选”之间的逻辑权衡。  关键质疑： 请审视文档中“状态可能丢失”这一设计在实际游戏场景（如文件传输或游戏进行中）的合理性。  输出： 请先输出一份简短的《架构实现路径》，指出你在实现“UDP 选主”与“WebSocket 状态同步”集成时的关键技术点，并确认你已理解本项目不是去中心化系统。  2. 全量自主开发约束拒绝占位符： 你必须独立完成 backend/（Java 17, Spring Boot 3）和 frontend/（Vue 3, Vite）的所有核心代码。  完整链路： 必须包含完整的 UDP 节点发现逻辑、选举算法实现、以及 STOMP 协议下的游戏状态广播机制。  持续生成： 严禁只完成部分功能。如果代码量大，请分段输出，但必须覆盖整个项目结构（core, net, app）。  3. 自动化测试驱动（Testing）核心测试逻辑： 必须编写单元测试来验证：选举逻辑： 给定不同 nodeId，能否按规则（version > nodeId）选出唯一 Host。  协议解析： 验证 UDP 的 HELLO 报文是否能被正确识别并触发节点更新。  状态分发： 模拟 WebSocket 消息能否通过 GameSession 正确广播至 /topic/game.state。  4. 技术栈合规性后端： Java 17, Spring Boot 3, WebSocket (STOMP), UDP Multicast。  前端： Vue 3, Pinia, Vue Router, stompjs。  # Design Principles极简主义： 移除所有鉴权、Token 和加密逻辑。  工程化： 虽然业务简化，但代码必须模块化，易于通过扩展 GameSession 实现类来增加新游戏。  # Ready to Start?
请先根据 ClassroomLAN v2 设计文档 确认你的理解，并开始第一步：业务逻辑审计。
